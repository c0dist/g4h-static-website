    Best practise for holding passwords in scripts
    I am sure all of us would have encountered clients leaving out shell/batch/perl scripts with username/password combinations in Clear text.

    Take an example of a shell script which runs every night without user intervention to transfer some files from the system to a FTP server and the FTP credentials are hardcoded in the script.

    I just want to understand what is the best practice and optimal solution to tackle clear text passwords in scripts.I know of few solutions which I came across through discussions and googling....but would love to know your experiences in such scenarios.

    In my case the application was written in VB.net which were taking username and password from a config file stored in installation directory in plain text.

    For this i suggested to put one Decryption module within application and encrypt the config file so that when application will get loaded it will collect the data from config file and decrypt it withing application and use it.

    however in scenario eby bro mentioned above similar solution can be used but still because of the nature of the scripting language algo for encryption/decryption will be available in plain text so again same thing will be thr, so it can be complicated by using session id as encryption key or something.(i am not sure if this is possible or not but seen something similer somewhere i guess Blowfish algo may help in this case.)
