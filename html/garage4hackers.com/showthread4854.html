    Let us start:

    Target OS : Windows XP
    Technique : Multithreaded

    First of all, the developer of an artificial life has to decide the final residing hive for the variant. I mean the folder and path where the malware will reside in the victim's computer.
    The path must be writable by lowest privileged user group.
    There are certain folders which are writable by non admin users also, these are like:

    Recycler
    drive:\Documents and Settings\All Users\Application Data
    Temp
    etc

    Some virus use Recycler (Recycle bin) to make their nests, this should be avoided. As making a folder or creating any file under Recycler will bring the suspicion of antivirus easily.

    We'll use following:
    "drive:\Documents and Settings\All Users\Application Data" As some of useful software's place their components and data their.
    Portion of this path is also saved into the ALLUSERSPROFILE variable into the environment and then changing directory to "Application Data".

    Note: Remember if ur malware will execute during booting stages, then do not use the environment variables as at that moment no environment exists upto some stages.

    Another Note: Do not use string values or constants at any place inside the code, instead define them in the beginning of the code. This will help u in many ways like later if u want to encrypt the values.

    Now its time to develop the USB infection circuit. There are several methods, I am going to discus one of the simplest methods of infection.

    In this method, first we'll recognize the drive type for removable type or remote type (remote drive / network drive). And if discovered, then, copy the necessary files to the target drive.

    The copying process must be repeated in an infinite loop with testified proper delay so that not to overwhelm the victim system.

    The process of copying is actually done in steps:

    1. Remove the S, H, R attributes of the files on target media (whether they exist or not)
    2. Delete the folder name and virus file (whether exists or not)
    3. Create the folder and copy files
    4. Set the attributes of the folder and files on target media to S, R, H.

    The USB infection routine will execute into its own thread, So we need to fix or prepare the required paths and stuff prior to this thread's execution.
    For examples we have prepared a path and returning this path from displacement circuit, So if we'll use that path, then we must execute the displacement circuit always prior to the dependent circuits.
    Note: Remember, try to avoid to code same stuff several times in worm/virus development, an artificial life developer must take care of all system resources and a good artificial life must not clogg the CPU and might not eat-up memory.
