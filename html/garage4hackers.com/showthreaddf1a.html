    simple-reverse-icmp-shel Reverse Shelll

    Source: http://www.pentestit.com/2011/04/15/...ontent=Twitter

    [I have mentioned about this stuffs once, found a good article and the script taught of sharing]

    Before we begin, lets know what a reverse shell is. A shell is a program/script that can execute commands on another computer across a computer network. A typical shell is where it opens up a connection and you connect to it. But on networks that are protected by firewalls or are behind NATs, this wont be successful as incoming traffic is mostly dis-allowed. This is where a reverse shell comes in handy as instead of you connecting to it, it connects back to your IP, bypassing weak security setups. But what happens if outgoing connections are allowed only on certain ports? This is where icmpsh comes in handy as it uses ICMP to connect back to you! What is ICMP? Internet Control Message Protocol (ICMP) is chiefly used by the operating systems to check if a requested service is not available or that a host or router could not be reached.

    In almost all networks that we have audited, ICMP is seldom disabled as it is one of the default ways to verify if a terminal is down or not. Network Monitoring tools like Whats Up Gold, etc. use ICMP. Back to icmpsh, it is a simple reverse ICMP shell with a win32 slave and a POSIX compatible master in C, Perl or Python. It has the following features:

    * Open source software – primarily coded by Nico, forked by the author.
    * Client/server architecture.
    * The master is portable across any platform that can run either C, Perl or Python code.
    * The target system has to be Windows because the slave runs on that platform only for now.
    * The user running the slave on the target system does not require administrative privileges.

    Only thing that might hurt you is – the slave (client) that runs on the target machine, works ONLY on Windows for now! But the good thing is that if you run the latest version of sqlmap (which we wrote about here) you can establish the out-of-band connections with icmpsh using ICMP or Metasploit using TCP! Not many dependencies for the master (but the Perl port has some) and none what so ever for the slave!

    Ah yes, when running the master, don’t forget to disable ICMP replies by the OS or the communication between the master and shell wont work!
