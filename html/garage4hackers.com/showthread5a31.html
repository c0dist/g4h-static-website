    Installing VMware / VMPlayer on Backtrack 5

    VMplayer was not able to get installed on backtrack 5.
    You may have similar problems in installing your drivers like say Nvidia drivers too.

    This goes as quick dirty fix to solve the VMplayer issue.
    First thing... Download the new VMplayer (File Name: VMware-Player-3.1.4-385536.i386.bundle)

    Secondly... You will have to download your kernel.
    No need to go to kernel.org or anywhere Just follow these instructions so that the kernel source will get downloaded and placed at right position.

    Make sure you are connected to internet while doing this.
    Pass following commands.

    prepare-kernel-sources
    cd /usr/src/linux
    cp -rf include/generated/* include/linux/

    Good to go ...

    Backtrack Linux Vmware 100% CPU mount.ntfs-3g

    Wmware takes 100% CPU
    mount.ntfs-3g 100% CPU
    Backtrack Vmware 100% CPU

    Hey any of you have experienced the following issues here is the workaround.

    Never move your Vmware vmx files form root Ext partition to any NTFS partition (backtrack 5) .
    If vmware tries to load Virtual machine files form NTFS partition then mount.ntfs-3g demon would utilize a high amount of CPU (bug??) making your system unusable .
    Considering backtrack 5 keep VM files on root partition itself that will make things fine.
