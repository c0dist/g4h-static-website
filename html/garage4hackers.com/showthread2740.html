    The final ev evaluates to eval. The eval runs the code which would be generated by for loop on iterating over variable n.

    Now we can easily replace eval by alert to instruct the javascript interpreter not to run the outcome of the block, but to pop up the outcome result in browser window. This is because most of the times the obfuscated code block is arranged at run time by the malicious code itself and then this de-obfuscated code is run by the eval. This whole drama is of-course to evade AVs and normal eye detection.
